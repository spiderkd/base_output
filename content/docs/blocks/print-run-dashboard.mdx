## Installation

```bash
npx shadcn add https://riso.dev/r/print-run-dashboard.json
```

## Components used

| Component | Count | Purpose |
|---|---|---|
| `StatCard` | 3 | Sheets printed (sparkline), avg misreg (filled), uptime |
| `BarChart` | 1 | Sheets per hour across 6 time slots |
| `Progress` | 3 | Print progress, ink remaining (halftone), registration accuracy |
| `Toggle` | 2 | Grain effect and misregistration controls |
| `Separator` | 2 | Double-rule and dotted section dividers |

## Customising

```tsx
// Replace the static data with live values:
<StatCard
  value={liveData.sheetsToday.toLocaleString()}
  label="Sheets Printed"
  trend={liveData.trendPct}
  sparkline={liveData.hourlyHistory}
/>

<Progress value={liveData.inkRemaining} label="Ink Remaining" variant="halftone" />
```

## Composition notes

The three stat cards create visual hierarchy: the center card uses `filled` (solid primary fill) to highlight the most important operational metric. The outer two use sparklines to show trend without needing extra chart space.

All components share the theme automatically via `--riso-primary` / `--riso-secondary` CSS variables from `RisoProvider`. No explicit color props needed.
