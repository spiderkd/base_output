## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `title` | `string` | `"Riso UI Version 2.0"` | Headline in PrintBlock header |
| `subtitle` | `string` | — | Subtitle below title |
| `date` | `string` | — | Date label in header |
| `author` | `string` | — | Author name in footer |
| `authorInitials` | `string` | — | Two-letter initials for Avatar |
| `tags` | `string[]` | — | Badge array below header |
| `status` | `"approved" \| "draft" \| "pending" \| "rejected"` | `"approved"` | Controls Stamp variant |
| `body` | `string` | — | Body paragraph text |

## Components used

| Component | Count | Purpose |
|---|---|---|
| `PrintBlock` | 1 | Solid primary header with grain texture |
| `Stamp` | 1 | Status overlapping the header corner |
| `Badge` | 3 | Tag row below header |
| `Separator` | 2 | Double-rule + dotted dividers |
| `Avatar` | 1 | Author initials circle |
| `Button` | 2 | Share and Read More CTAs |

## Composition notes

The `Stamp` is `position: absolute` with `top: 12px; right: 16px; z-index: 10` — it deliberately overlaps the `PrintBlock` header, simulating a rubber stamp applied after the document was printed. The rotation (−2°) reinforces the hand-stamped feel.

The `PrintBlock` header has `paddingRight: 80px` to leave room for the stamp so text never runs underneath it.

The `status` prop maps directly to the `Stamp` `variant` prop — `"approved" | "draft" | "pending" | "rejected"` — using each variant's configured ink colors automatically.

The two `Separator` components serve different visual weights: the first (double-rule, heavier) divides the tags from the body; the second (dotted) separates the body from the author footer.
