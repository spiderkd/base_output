## Props

The examples above cover the most common ways to use `Breadcrumb`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `items` | `BreadcrumbItem[]` | - | Ordered breadcrumb items with labels and optional actions. |
| `maxItems` | `number` | `4` | Configures `maxItems` for the component. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

The separator is two SVG `<line>` elements: a 2.5px primary diagonal slash (`10,2 → 4,18`) and a 1px secondary slash offset 2px right (`12,4 → 6,20`).

When collapsed, the ellipsis button renders three 4×4 squares in alternating primary, secondary, and overlap colors. Clicking expands the full path.

Active (last) item has a 2px primary underline. Parent items are secondary color with no underline.

That restraint is what keeps `Breadcrumb` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
