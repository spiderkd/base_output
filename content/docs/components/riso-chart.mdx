## Props

The examples above cover the most common ways to use `RisoChart`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `data` | `ChartSegment[]` | - | Segment data for the pie or donut chart. |
| `size` | `number` | - | Overall SVG size in pixels. |
| `donut` | `boolean` | - | Uses a donut center instead of a full pie. |
| `showLegend` | `boolean` | - | Shows the legend below the chart. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

Each segment is an SVG arc path with `style={{ mixBlendMode: "multiply" }}`. Where two segments share a boundary, their colors multiply — primary pink × secondary teal = overlap purple, automatically, from the blend mode. No explicit overlap color is set.

The `feTurbulence` grain filter is applied per segment giving each a slightly different texture. Hover lifts a segment with `drop-shadow(4px 4px 0 var(--riso-overlap))`.

That restraint is what keeps `RisoChart` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
