## Props

The examples above cover the most common ways to use `Tooltip`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `content` | `React.ReactNode` | - | Tooltip content shown on hover or focus. |
| `placement` | `TooltipPlacement` | `"top"` | Position of the tooltip relative to the trigger. |
| `children` | `React.ReactElement` | - | Single trigger element wrapped by the tooltip. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

The tooltip body is a solid primary ink rectangle — no border-radius, no blur, no fade-in. It appears in 80ms and disappears in 80ms because riso ink doesn't fade.

The arrow pointer is two overlapping SVG `<polygon>` elements: a secondary shadow triangle offset `2px right, 2px down` behind the primary triangle. This gives the pointer the same misregistration depth as every other component.

The `2px` secondary shadow behind the main tooltip box is lighter than the button's `4px` shadow — tooltips sit closer to the surface than interactive elements.

That restraint is what keeps `Tooltip` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
