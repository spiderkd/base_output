## Props

The examples above cover the most common ways to use `PlateRegistration`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `variant` | `RegistrationVariant` | `"target"` | Registration mark style to render. |
| `corner` | `RegistrationCorner` | `"tl"` | Corner used when the corner variant is selected. |
| `size` | `number` | `48` | Overall marker size in pixels. |
| `primaryOpacity` | `number` | `0.8` | Opacity of the primary plate. |
| `secondaryOpacity` | `number` | `0.55` | Opacity of the secondary plate. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

In real riso printing, registration marks are printed in both ink passes so the operator can check alignment. The offset between the two renderings shows the misregistration. This component decorates your UI with those same marks — a nod to the physical print process.

All marks use two-layer SVG: primary ink (full weight) + secondary ink (lighter, offset 2px right/down). The `full-bleed` variant uses `position: absolute; inset: 0` and places a `CornerMark` at each corner.

That restraint is what keeps `PlateRegistration` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
