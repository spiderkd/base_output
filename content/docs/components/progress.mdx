## Props

The examples above cover the most common ways to use `Progress`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `value` | `number` | - | Progress value from `0` to `100`. |
| `label` | `string` | - | Optional label shown above the bar. |
| `showValue` | `boolean` | `true` | Shows the numeric percentage label. |
| `variant` | `"default" \| "ink-bleed" \| "halftone"` | `"default"` | Visual treatment for the fill. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Variant reference

| Variant     | What it adds                                                                                |
| ----------- | ------------------------------------------------------------------------------------------- |
| `default`   | Solid primary fill, white halftone dot texture on fill, ink-bleed tip at leading edge       |
| `halftone`  | Radial dot grid (`4 × 4 px`) at 20% opacity over the unfilled track region                  |
| `ink-bleed` | Gradient from `--riso-primary` → `--riso-overlap` at the leading edge, `mix-blend-multiply` |

## Design notes

The track is an 18 px tall rectangle with `paper` background and a 2 px primary outline. A 3 px secondary block sits behind it at 35% opacity — the hard shadow motif used throughout the system.

Fill width animates via CSS `transition: width 400ms cubic-bezier(0.4, 0, 0.2, 1)` — value changes play smoothly whether driven by state or a live data feed. The bleed tip only renders when `0 < value < 100` to avoid overflowing the track edges.

That restraint is what keeps `Progress` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
