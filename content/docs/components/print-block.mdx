## Props

The examples above cover the most common ways to use `Print Block`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `ink` | `"primary" \| "secondary" \| "overlap"` | `"primary"` | Ink channel used for the printed block. |
| `rotate` | `number` | `0` | Rotation angle in degrees. |
| `children` | `React.ReactNode` | - | Content placed inside the block. |

## Design notes

Print Block is the simplest riso component — a single flat ink color with grain texture applied via `feTurbulence`. It has **no border, no outline, no hard shadow**. The grain filter is the only texture.

This simulates what a single ink pass looks like on paper: flat, opaque, slightly irregular at the edges from the drum. Content inside should be white or very light.

The `rotate` prop accepts a small degree value (typically -2 to +2°) to simulate the block being hand-placed on the paper before print — a common riso design convention.

That restraint is what keeps `Print Block` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
