## Props

The examples above cover the most common ways to use `Stepper`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `steps` | `Step[]` | - | Ordered step definitions. |
| `currentStep` | `number` | - | Zero-based active step index. |
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | Layout direction for the stepper. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Node states

| Status                            | Visual                                                         |
| --------------------------------- | -------------------------------------------------------------- |
| `complete` (`idx < currentStep`)  | Solid primary fill, white checkmark, secondary shadow circle   |
| `current` (`idx === currentStep`) | Primary outline, secondary shadow, inner filled dot            |
| `upcoming` (`idx > currentStep`)  | Dashed `strokeDasharray="3 2.5"` primary circle at 50% opacity |

## Design notes

Connector lines between steps use the double-rule pattern: 2px primary + 1px secondary offset 3px. Completed connectors are solid at full opacity. Upcoming connectors drop to 35% opacity. No transition on the connector — it changes immediately when `currentStep` updates, consistent with the stamp's instantaneous quality.

That restraint is what keeps `Stepper` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
