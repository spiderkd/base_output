## Props

The examples above cover the most common ways to use `Heatmap`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `data` | `number[][]` | - | 2D array of normalized values from `0` to `1`. |
| `rowLabels` | `string[]` | - | Labels shown along the rows. |
| `colLabels` | `string[]` | - | Labels shown across the columns. |
| `cellSize` | `number` | `36` | Cell size in pixels. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

This is the defining riso-unique data display. Each cell gets a unique SVG `<pattern>` with a dot radius mapped from the value (0.8px â†’ 5px). At high values dots overlap, creating the overlap ink color naturally â€” the paper shows through less.

Hover shows the numeric percentage (0â€“100) centered in the cell and jumps the outline to 2px. Hover shadow is a secondary-color div offset `2px right, 2px down` at 35% opacity.

That restraint is what keeps `Heatmap` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
