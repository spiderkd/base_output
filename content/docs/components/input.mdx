## Props

The examples above cover the most common ways to use `Input`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `label` | `string` | - | Optional field label. |
| `error` | `string` | - | Error message shown below the field. |
| `hint` | `string` | - | Helper copy shown when there is no error. |
| `halftone` | `boolean` | `false` | Adds a halftone paper texture behind the input. |

`Input` also accepts standard HTML input props such as `value`, `defaultValue`, `onChange`, `placeholder`, `type`, and `disabled`.

## Design notes

The misregistration effect is achieved with an absolutely-positioned `div` in the secondary ink color, offset `2px right, 2px down` behind the input frame. On focus, the outline shifts from secondary → primary color, simulating the second ink plate snapping into sharper registration.

There is intentionally **no border-radius**. All riso components use `0px` corners — the only exception is pill badges.

That restraint is what keeps `Input` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
