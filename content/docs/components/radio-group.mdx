## Props

The examples above cover the most common ways to use `RadioGroup`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `options` | `RadioOption[]` | - | Radio options rendered by the group. |
| `value` | `string` | - | Controlled selected value. |
| `defaultValue` | `string` | - | Initial selected value for uncontrolled usage. |
| `onChange` | `(value: string) => void` | - | Called when the selected option changes. |
| `label` | `string` | - | Optional group label. |
| `orientation` | `"vertical" \| "horizontal"` | `"vertical"` | Layout direction for the options. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

The radio indicator is two concentric SVG circles — the same geometry as the Avatar component. Outer circle: 2px primary border. Inner: paper background idle, floods with primary on select. A secondary shadow circle (1.5px, offset 2.5px) sits behind the outer ring.

RadioGroup uses circles rather than the squares used everywhere else in the system. Circles are reserved for radio buttons and avatars — the only rounded shapes in Riso UI (aside from pills and stamps).

That restraint is what keeps `RadioGroup` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
