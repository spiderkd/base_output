## Props

The examples above cover the most common ways to use `StatCard`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `value` | `string \| number` | - | Main metric value shown in the card. |
| `label` | `string` | - | Metric label shown under the value. |
| `trend` | `number` | - | Numeric trend value, positive or negative. |
| `trendLabel` | `string` | - | Short label paired with the trend value. |
| `sparkline` | `number[]` | - | Mini trend series rendered in the corner. |
| `filled` | `boolean` | `false` | Uses a filled ink background. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## The ghost text effect

The display number is rendered twice: the primary value at full opacity in `var(--riso-primary)`, and a ghost copy at 35% opacity in `var(--riso-secondary)` positioned `2px right, 2px down` using `position: absolute`. This creates the misregistration effect on the number itself.

The sparkline follows the same two-layer rule: a secondary ink polyline at 1.5px sits offset behind the primary 2px polyline. Both use `strokeLinecap="square"` — no rounded ends.

## Design notes

That restraint is what keeps `StatCard` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
