## Props

The examples above cover the most common ways to use `InkBlotLoader`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `size` | `LoaderSize` | `"md"` | Controls the overall component scale. |
| `label` | `string` | - | Optional label shown under the animation. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## How it works

Two organic SVG `<path>` elements animate between three keyframe shapes using CSS `d: path(...)` morphing — no JavaScript in the animation loop.

The primary blob animates on a 2.4s cycle. The secondary blob on a 2.8s cycle with a 0.4s delay — so they are never perfectly in phase, creating a continuously shifting overlap region. Where they overlap, `mix-blend-mode: multiply` produces `var(--riso-overlap)` — the third ink color — without any explicit color being set.

`prefers-reduced-motion: reduce` stops both animations, leaving the blobs in their default positions.

That restraint is what keeps `InkBlotLoader` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
