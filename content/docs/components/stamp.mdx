## Props

The examples above cover the most common ways to use `Stamp`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `label` | `string` | - | Primary text shown inside the stamp. |
| `sublabel` | `string` | - | Secondary text shown beneath the label. |
| `variant` | `StampVariant` | `"default"` | Stamp border and fill style. |
| `size` | `number` | `100` | Overall stamp size in pixels. |
| `rotate` | `number` | `-2` | Rotation angle in degrees. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

The stamp is built entirely in SVG — two concentric `<circle>` elements with `stroke-dasharray` to create the serrated border. The outer circle uses the main ink color; the inner circle uses a complementary ink at lower opacity. Their `strokeDasharray` values are offset so the dashes don't align — the inner and outer teeth interlock like a real stamp die.

The `filter: url(#riso-bleed-soft)` makes the stroke slightly ink-bleed at its edges — a soft version of the same bleed filter used on avatar circles.

The `rotate` prop (default varies by variant) applies a small CSS rotation to the container, making the stamp look hand-placed. Rotation is applied _outside_ the SVG so the drop shadow also rotates with it.

That restraint is what keeps `Stamp` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
