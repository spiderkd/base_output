## Props

The examples above cover the most common ways to use `MisregTextReveal`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `text` | `string` | - | Text content animated by the reveal effect. |
| `as` | `"h1" \| "h2" \| "h3" \| "p" \| "span"` | `"h2"` | HTML tag used to render the text. |
| `size` | `number` | `36` | Font size in pixels. |
| `animateOnMount` | `boolean` | `true` | Runs the animation immediately on mount. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## How the animation works

Two absolutely-positioned `<Tag>` elements share the same text.

The primary layer (`color: var(--riso-primary)`) slides in from `translateX(-24px)` over 400ms. The secondary layer (`color: var(--riso-secondary)`, opacity 0.42) follows 80ms later with an overshoot to `translate(6px, 6px)` before settling to `translate(2px, 2px)` via `cubic-bezier(0.22, 1, 0.36, 1)`.

On hover: the secondary layer separates further to `translate(6px, 4px)` — simulating plate misalignment — then snaps back on mouse leave.

When `animateOnMount: false`, `IntersectionObserver` triggers the animation when 30% of the element is visible.

## Design notes

That restraint is what keeps `MisregTextReveal` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
