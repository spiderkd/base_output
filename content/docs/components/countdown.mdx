## Props

The examples above cover the most common ways to use `Countdown`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `targetDate` | `Date` | - | Target date the countdown runs toward. |
| `seconds` | `number` | - | Alternative countdown length in seconds. |
| `onComplete` | `() => void` | - | Called when the countdown reaches zero. |
| `showLabels` | `boolean` | `true` | Configures `showLabels` for the component. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

Each digit slot is a 72 × 88 px card with `paper` background and a 4 px hard shadow in secondary.

On every tick two layers animate:

- **Primary** — slides in from `translateY(-100%)` to `0` over 220 ms
- **Secondary ghost** — follows 60 ms later, settles at `translate(3px, 3px)`, giving the misreg offset

The colon separator uses the double-rule motif: two 6 × 6 px squares, primary on top, secondary offset 2 px down-right at 55% opacity.

Days segment only renders when `remaining >= 86400` — below that it auto-hides to keep the layout compact.

That restraint is what keeps `Countdown` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
