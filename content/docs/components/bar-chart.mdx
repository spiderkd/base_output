## Props

The examples above cover the most common ways to use `BarChart`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `data` | `BarDatum[]` | - | Bar data rendered in order across the chart. |
| `width` | `number` | `400` | Overall chart width. |
| `height` | `number` | `180` | Overall chart height. |
| `margin` | `Partial<Margin>` | - | Configures `margin` for the component. |
| `showValues` | `boolean` | `true` | Configures `showValues` for the component. |
| `showGrid` | `boolean` | `true` | Configures `showGrid` for the component. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

**No gradients, no hachure.** Each bar is a solid `<rect>` in a flat ink color. This is the core discipline of the riso aesthetic — ink is opaque and flat, not decorative.

**Double-rule X axis.** The baseline is two lines: 2px primary and 1px secondary at `y + 3`. This is the same pattern as the Separator component.

**Dotted grid lines.** Instead of solid grey lines, horizontal grid markers are dashed secondary-color lines at 35% opacity.

**Hover shadow.** On hover, a same-size rectangle in the opposite ink color appears at `+3px, +3px` offset behind the bar at 38% opacity. The cursor becomes `pointer`. No animation — it appears instantly (120ms would feel slow for a chart).

**Halftone overlay on fill.** Every bar has a `radial-gradient` dot pattern at 22% white opacity overlaid — very subtle on screen, simulates the paper showing through the ink.

That restraint is what keeps `BarChart` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
