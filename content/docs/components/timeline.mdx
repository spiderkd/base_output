## Props

The examples above cover the most common ways to use `Timeline`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `items` | `TimelineItem[]` | - | Timeline items rendered along the vertical spine. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Animated reveal

Items enter one by one on mount — useful for onboarding flows or dashboards.

<PreviewContainer
  componentName="timeline"
  code={`// Drive visible count with a useEffect interval, render sliced items
const [visible, setVisible] = React.useState(0)

useEffect(() => {
if (visible >= items.length) return
const t = setTimeout(() => setVisible(v => v + 1), 350)
return () => clearTimeout(t)
}, [visible])

<Timeline items={items.slice(0, visible)} />`}

## Item status reference

| Status     | Node                            | Content card                                        |
| ---------- | ------------------------------- | --------------------------------------------------- |
| `complete` | Solid primary + white checkmark | Light tint outline                                  |
| `current`  | Primary outline + inner dot     | Full `drop-shadow(3px 3px 0 var(--riso-secondary))` |
| `upcoming` | Dashed primary outline          | Reduced opacity label                               |

## Design notes

The spine is two vertical lines: 2 px primary + 1 px secondary offset 4 px right at 70% opacity — the double-rule motif applied vertically. Spine nodes are SVG circles with a secondary shadow circle offset 2.5 px down-right.

Date labels sit left of the spine (`text-align: right`, `white-space: nowrap`) in secondary uppercase 8 px tracking. Content cards sit right. `current` cards get the full 3 px hard shadow. Label opacity drops to 50% for `upcoming` items to signal they haven't happened yet without hiding the information.

That restraint is what keeps `Timeline` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
