## Props

The examples above cover the most common ways to use `Dialog`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `open` | `boolean` | - | Controls whether the dialog is visible. |
| `onClose` | `() => void` | - | Called when the dialog should close. |
| `title` | `string` | - | Heading shown at the top of the component. |
| `description` | `string` | - | Optional supporting text. |
| `children` | `React.ReactNode` | - | Custom dialog body content. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

The secondary ink layer starts at `translate(12px, 12px)` and springs to `translate(4px, 4px)` with `cubic-bezier(0.22, 1, 0.36, 1)` over 280ms on open — simulating the second riso plate dropping into rough registration.

The backdrop is a halftone dot pattern in the overlap color at 18% opacity. No `backdrop-filter: blur()`. There is no blur anywhere in Riso UI.

Uses the native `<dialog>` element. Focus is trapped automatically by the browser. `Escape` closes it.

That restraint is what keeps `Dialog` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
