## Props

The examples above cover the most common ways to use `InkCursor`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `maxDots` | `number` | - | Maximum dots kept in the cursor trail. |
| `dotLife` | `number` | - | How long each dot stays visible, in milliseconds. |
| `dotRadius` | `number` | - | Base radius of each cursor dot. |
| `density` | `number` | - | How frequently trail dots are spawned. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Layout placement

```tsx
// app/layout.tsx
import { InkCursor } from "@/registry/riso/ui/ink-cursor";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        <InkCursor /> {/* fixed canvas, pointer-events: none */}
        {children}
      </body>
    </html>
  );
}
```

## How it works

A `<canvas>` fixed to `position: fixed; inset: 0` with `pointer-events: none; mix-blend-mode: multiply`. It never blocks interaction.

Dots deposit on `mousemove` if the cursor has moved at least `density` px from the last point. Each dot alternates between primary and secondary ink. Position has `±1.5px` random offset simulating ink spread irregularity.

Each dot's radius grows slightly as it ages (`r = baseR × (1 + age × 0.3)`) — simulating ink bleeding outward. Alpha fades from 0.8 to 0.

`ctx.globalCompositeOperation = "multiply"` means overlapping primary + secondary dots produce the overlap color automatically — no explicit color set for it.

That restraint is what keeps `InkCursor` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
