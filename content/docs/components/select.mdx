## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `options` | `SelectOption[]` | — | `{ value, label, disabled? }[]` |
| `value` | `string` | — | Controlled value |
| `defaultValue` | `string` | — | Uncontrolled default |
| `onChange` | `(value: string) => void` | — | Change handler |
| `placeholder` | `string` | `"Select…"` | Placeholder text |
| `label` | `string` | — | Field label |
| `disabled` | `boolean` | `false` | Disables the entire select |

## Design notes

The trigger uses the same misregistration shadow as `Input` — a secondary-color div offset `2px right, 2px down`, opacity jumping from `0.22` (idle) to `0.35` (open).

The chevron is four SVG `<line>` elements: two primary arms (2px, square cap) + two secondary arms (1px, offset 1.5px, opacity 0.6) — the same chevron used in Accordion and Tabs.

The dropdown list gets `drop-shadow(4px 4px 0 var(--riso-secondary))`. Selected option: solid primary fill. Hover: 15% secondary tint. No border-radius anywhere.
