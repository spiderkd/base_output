## Props

The examples above cover the most common ways to use `Select`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `options` | `SelectOption[]` | - | Available options rendered in the dropdown. |
| `value` | `string` | - | Controlled selected value. |
| `defaultValue` | `string` | - | Initial selected value for uncontrolled usage. |
| `onChange` | `(value: string) => void` | - | Called when the selected option changes. |
| `placeholder` | `string` | `"Select…"` | Text shown when nothing is selected. |
| `label` | `string` | - | Optional field label. |
| `disabled` | `boolean` | `false` | Disables the trigger and option selection. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

The trigger uses the same misregistration shadow as `Input` — a secondary-color div offset `2px right, 2px down`, opacity jumping from `0.22` (idle) to `0.35` (open).

The chevron is four SVG `<line>` elements: two primary arms (2px, square cap) + two secondary arms (1px, offset 1.5px, opacity 0.6) — the same chevron used in Accordion and Tabs.

The dropdown list gets `drop-shadow(4px 4px 0 var(--riso-secondary))`. Selected option: solid primary fill. Hover: 15% secondary tint. No border-radius anywhere.

That restraint is what keeps `Select` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
