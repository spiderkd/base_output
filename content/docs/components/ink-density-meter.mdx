## Props

The examples above cover the most common ways to use `InkDensityMeter`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `value` | `number` | - | Fill value from `0` to `100`. |
| `label` | `string` | - | Optional label shown above the meter. |
| `height` | `number` | `160` | Configures `height` for the component. |
| `width` | `number` | `44` | Configures `width` for the component. |
| `showValue` | `boolean` | `true` | Shows the numeric reading under the meter. |
| `animated` | `boolean` | `true` | Animates fill changes. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## How the density fill works

The meter is divided into 12 horizontal bands. Each band has a unique SVG `<pattern>` with dot radius growing from `0.8px` (bottom, dense end for low values) to `5px` (top, dense end for high values). Only bands within the fill level render.

On value change, the fill animates via `requestAnimationFrame` with an `ease-out-cubic` curve over 600ms. Three tick marks at 25%, 50%, 75% use the double-rule pattern (primary + secondary offset).

## Design notes

That restraint is what keeps `InkDensityMeter` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
