## Props

The examples above cover the most common ways to use `Pagination`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `total` | `number` | - | Total number of items in the collection. |
| `page` | `number` | - | Current page number. |
| `perPage` | `number` | `10` | Items shown per page. |
| `onChange` | `(page: number) => void` | - | Called with the next page number. |
| `siblings` | `number` | `1` | Pages shown on each side of the active page. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

Current page: solid `var(--riso-primary)` fill with `drop-shadow(2px 2px 0 var(--riso-secondary))` — the small stamp effect. Other pages: transparent background, primary text, light primary outline.

Prev/Next chevrons: the same four-line SVG pattern as Accordion and Select — primary 2px arms + secondary 1px offset arms at 60% opacity.

Ellipsis: `···` in secondary color. The `siblings` prop controls how many pages appear around the current page before collapsing.

That restraint is what keeps `Pagination` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
