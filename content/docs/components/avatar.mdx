## Props

The examples above cover single avatars, size changes, and grouped layouts. Start with the core avatar props, then use `AvatarGroup` when you want a stacked presentation.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `src` | `string` | - | Image source for the avatar artwork |
| `alt` | `string` | `""` | Accessible alt text for the image |
| `initials` | `string` | - | Fallback initials shown when no image is provided |
| `size` | `"sm" \| "md" \| "lg" \| "xl"` | `"md"` | Controls the avatar scale |
| `variant` | `"default" \| "stamp" \| "filled"` | `"default"` | Changes the border and fill treatment |

## AvatarGroup props

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `children` | `React.ReactNode` | - | Avatars rendered in the overlapping group |

Both `Avatar` and `AvatarGroup` accept the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

That restraint is what keeps `Avatar` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
