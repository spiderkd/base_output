## Props

The examples above cover the most common ways to use `SidebarNav`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `sections` | `NavSection[]` | - | Navigation sections and their child items. |
| `activeId` | `string` | - | Currently active navigation item id. |
| `onSelect` | `(id: string) => void` | - | Called when a navigation item is selected. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Item definition

```tsx
interface NavItem {
  id: string;
  label: string;
  href?: string; // optional — used for <a> href
  icon?: ReactNode; // optional inline SVG
  badge?: string | number; // optional badge value (right-aligned)
}
```

## Design notes

Active item: a 4px solid primary left strip grows from `width: 0` via CSS transition, plus 10% primary tint background. Uses `position: absolute; left: 0` — not CSS `border-left`, which would shift the layout.

Hover: a 2px secondary left strip fades in with 12% secondary tint background.

Section headers sit above a double-rule at reduced opacity (secondary 40% top, primary 20% bottom) — quieter than the full Separator component.

Badge values use solid primary fill if active, solid secondary fill otherwise — 8px uppercase, flush right.

That restraint is what keeps `SidebarNav` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
