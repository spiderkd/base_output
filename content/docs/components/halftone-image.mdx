## Props

The examples above cover the most common ways to use `HalftoneImage`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `src` | `string` | - | Image source converted into a halftone treatment. |
| `alt` | `string` | - | Accessible alt text for the image. |
| `width` | `number \| string` | `"100%"` | Configures `width` for the component. |
| `height` | `number \| string` | `"auto"` | Configures `height` for the component. |
| `dotSize` | `number` | `3` | Size of each halftone dot. |
| `angle` | `number` | `15` | Screen angle used for the dot pattern. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## How it works

Three layers render over the source image:

1. The image filtered through `feColorMatrix(saturate, 0)` + `feComponentTransfer(discrete)` to posterize
2. A rotated `<pattern>` of circles in the primary ink color
3. A second offset `<pattern>` in secondary ink at `angle + 30°`, 45% opacity

Where both layers overlap, `mix-blend-mode: multiply` produces the overlap color — exactly as if two ink passes were physically made. A grain overlay (7% opacity SVG data URI) finalizes the printed-paper texture.

That restraint is what keeps `HalftoneImage` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
