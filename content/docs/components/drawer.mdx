## Props

The examples above cover the most common ways to use `Drawer`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `open` | `boolean` | - | Controls whether the drawer is visible. |
| `onClose` | `() => void` | - | Called when the drawer should close. |
| `side` | `DrawerSide` | `"right"` | Screen edge the drawer slides in from. |
| `title` | `string` | - | Heading shown at the top of the component. |
| `description` | `string` | - | Optional supporting text. |
| `children` | `React.ReactNode` | - | Custom drawer body content. |
| `width` | `number \| string` | `360` | Drawer width in pixels or any CSS length. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

The secondary shadow layer enters with a 60ms delay after the primary panel, creating the "trailing misregistration" effect. At rest it settles to `translate(4px, 4px)` (or the directional equivalent for each side) via `cubic-bezier(0.22, 1, 0.36, 1)`.

The halftone backdrop (`radial-gradient` of overlap-color dots at 5px spacing, 20% opacity) replaces `backdrop-filter: blur()`. There is no blur anywhere in Riso UI.

The header uses the PrintBlock treatment: solid primary fill, ghost text `textShadow: "1.5px 1.5px 0 var(--riso-secondary)"`.

That restraint is what keeps `Drawer` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
