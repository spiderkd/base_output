## Props

The examples above cover the most common ways to use `Calendar`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `value` | `Date` | - | Controlled selected date. |
| `defaultValue` | `Date` | - | Initial selected date for uncontrolled usage. |
| `onChange` | `(date: Date) => void` | - | Called when the value changes. |
| `events` | `CalendarEvent[]` | `[]` | Dates or events highlighted in the calendar grid. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

Selected date: solid `var(--riso-primary)` fill — flat ink square covering the cell. No border-radius.

Today (if not selected): 2px primary outline with `drop-shadow(2px 2px 0 var(--riso-secondary))`.

Event indicators are 5×5 squares (not dots) below the date number, with a negative `2px` bottom margin making them bleed past the cell boundary.

Month header lives in a PrintBlock (solid primary fill). Day-of-week headers are secondary uppercase labels. Weekends are secondary color numbers.

That restraint is what keeps `Calendar` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
