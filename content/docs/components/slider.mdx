## Props

The examples above cover the most common ways to use `Slider`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `min` | `number` | `0` | Minimum slider value. |
| `max` | `number` | `100` | Maximum slider value. |
| `step` | `number` | `1` | Step size between values. |
| `value` | `number` | - | Controlled slider value. |
| `defaultValue` | `number` | `0` | Initial value for uncontrolled usage. |
| `onChange` | `(value: number) => void` | - | Called when the slider value changes. |
| `showValue` | `boolean` | `true` | Shows the current numeric value. |
| `showTicks` | `boolean` | `false` | Shows tick marks along the track. |
| `label` | `string` | - | Optional slider label. |
| `disabled` | `boolean` | `false` | Disables interaction. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

The track is a pure SVG — two `<line>` elements (primary 4px + secondary 1.5px) offset `2px` apart, simulating a double-rule misregistration. The thumb is a `<rect>` with a hard shadow behind it in the primary color. The native `<input type="range">` sits invisibly over the SVG and handles all browser interaction.

That restraint is what keeps `Slider` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
