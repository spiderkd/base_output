## Props

The examples above cover the most common ways to use `LineChart`. Start with the default example, then layer in the props below when you need more control.

| Prop | Type | Default | Description |
| -------------- | ------------------------- | ----------- | ------------------------------- |
| `data` | `DataPoint[]` | - | Point data rendered along the line. |
| `height` | `number` | `180` | Configures `height` for the component. |
| `showArea` | `boolean` | `true` | Fills the area under the line. |
| `showGrid` | `boolean` | `true` | Shows chart grid lines. |
| `showDots` | `boolean` | `true` | Shows point markers on each data value. |

This component also accepts the shared `theme`, `primary`, `secondary`, `overlap`, and `paper` overrides.

## Design notes

The area fill is a `<polygon>` filled with a `<pattern>` of 1px circles at 4px spacing — a halftone wash rather than a transparent alpha fill.

Data points are 6×6 `<rect>` squares, not circles. All riso chart data points are squares consistently.

On hover, a dashed vertical guide appears at the hovered x-position and the value floats above the data point in primary color.

That restraint is what keeps `LineChart` feeling printed instead of purely digital: the offsets stay small, the edges stay sharp, and the ink contrast does most of the visual work.
